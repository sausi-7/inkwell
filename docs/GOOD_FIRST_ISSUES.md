# Good first issues

The live list lives on GitHub — that way claiming one, discussing, and tracking progress all happens in one place.

👉 **Browse open starter tickets:**
[github.com/sausi-7/inkwell/issues?q=is:open+label:"good first issue"](https://github.com/sausi-7/inkwell/issues?q=is%3Aopen+label%3A%22good+first+issue%22)

Or filter by category:

- **[scanner](https://github.com/sausi-7/inkwell/issues?q=is%3Aopen+label%3A%22scanner%22)** — new platform integration (HN, Product Hunt, Dev.to, …)
- **[exporter](https://github.com/sausi-7/inkwell/issues?q=is%3Aopen+label%3A%22exporter%22)** — new output target (Notion, Airtable, Slack, …)
- **[persona](https://github.com/sausi-7/inkwell/issues?q=is%3Aopen+label%3A%22persona%22)** — contribute a persona template to the marketplace
- **[help wanted](https://github.com/sausi-7/inkwell/issues?q=is%3Aopen+label%3A%22help+wanted%22)** — bigger features, maintainer would love help

## Claiming one

Comment on the issue — "I'd like to take this" — and a maintainer will assign. No obligation if life happens; just drop a note if you step away so someone else can pick it up.

## Adding a new starter ticket

If you spot a gap, open an issue using one of the [issue templates](https://github.com/sausi-7/inkwell/issues/new/choose). Good starter tickets are:

- Scoped enough to ship in one sitting
- Well-documented (files to touch, acceptance criteria)
- Match an existing protocol (scanner / exporter / persona) so the path is clear

For direction on where the project is headed, see **[ROADMAP.md](ROADMAP.md)**.
